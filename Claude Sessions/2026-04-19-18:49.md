### Fixed Glow for Files in Subdirectories

The original glow `.desktop` launcher used `%U` which broke on paths containing spaces
(e.g. `Claude Sessions/`). Fixed by switching to `%F` and quoting the argument as `"$1"`.
Works at any directory depth.

```bash
# ~/.local/share/applications/glow.desktop — Exec line fix:
# Before:
Exec=bash -c 'eval "$(...brew shellenv)" && glow %U; read ...'
# After:
Exec=bash -c 'eval "$(...brew shellenv)" && glow "$1"; read ...' -- %F
```
